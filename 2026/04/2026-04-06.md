## Java vs JavaScript — 뭐가 다를까?

겉보기엔 문법이 비슷하지만, 본질적으로 꽤 다른 언어라는 걸 알게 됐다.

| 항목 | JavaScript | Java |
|------|-----------|------|
| 타입 시스템 | 동적 타입 (런타임에 결정) | 정적 타입 (컴파일 시 결정) |
| 실행 환경 | 브라우저 / Node.js | JVM (Java Virtual Machine) |
| 변수 선언 | `let`, `const`, `var` | `int`, `String`, `boolean` 등 |
| 클래스 | ES6+에서 추가 (문법적 설탕) | 처음부터 클래스 기반 OOP |
| 컴파일 | 인터프리터 (즉시 실행) | 컴파일 후 `.class` 파일 생성 |
| null 처리 | `undefined` / `null` 모두 존재 | `null` 만 존재 |

## JDK, JRE, JVM 정리

처음 Java를 설치할 때 헷갈렸던 용어들인데, 이렇게 정리하니 한 번에 이해됐다.

- **JVM**: Java 바이트코드를 OS 위에서 실행하는 가상 머신. "Write Once, Run Anywhere"의 핵심
- **JRE**: JVM + 실행에 필요한 라이브러리. 실행만 할 때 필요
- **JDK**: JRE + 컴파일러(`javac`) + 개발 도구. 개발할 때 필요

> 프론트엔드 비유로 이해하면: JVM은 브라우저, JRE는 브라우저+기본 API, JDK는 브라우저+Node.js+빌드툴 세트라고 생각하면 된다.

## 정적 타입 시스템

JavaScript에서는 `let x = 5` 이후 `x = "hello"` 가 가능하지만, Java는 컴파일 시점에 타입 오류를 잡아준다. TypeScript를 쓰는 이유와 완전히 같은 맥락이라는 게 인상적이었다.

```java
int x = 5;
x = "hello"; // ❌ 컴파일 에러! int에 String 할당 불가

String name = "동빈";
name = 100; // ❌ 컴파일 에러!
```

## 실습 코드

### Hello World

```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

처음엔 왜 이렇게 많이 써야 하나 싶었는데, 이유가 있었다.
- `public class HelloWorld` → 모든 Java 코드는 클래스 안에 있어야 함
- `public static void main(String[] args)` → 프로그램의 진입점
- `System.out.println` → JS의 `console.log` 와 동일

### 변수와 자료형

```java
public class Variables {
    public static void main(String[] args) {
        int age = 28;
        long bigNumber = 10000000000L;
        double price = 19.99;
        String name = "동빈";
        String greeting = "안녕하세요, " + name + "님!";
        String formatted = String.format("이름: %s, 나이: %d", name, age);
        boolean isLearning = true;

        System.out.println(greeting);   // 안녕하세요, 동빈님!
        System.out.println(formatted);  // 이름: 동빈, 나이: 28
    }
}
```

## 핵심 포인트

1. **Java는 정적 타입 언어** — 변수 선언 시 타입 명시 필수. TypeScript랑 개념이 같다.
2. **모든 코드는 클래스 안에** — 파일 최상위에 코드 작성 불가
3. **`main` 메서드가 진입점** — `public static void main(String[] args)` 부터 실행 시작
4. **`String`은 대문자** — 기본 타입(`int`, `boolean`)은 소문자, `String`은 클래스라 대문자
5. **세미콜론 필수** — JS에서는 선택이지만 Java는 반드시 `;` 붙여야 함
